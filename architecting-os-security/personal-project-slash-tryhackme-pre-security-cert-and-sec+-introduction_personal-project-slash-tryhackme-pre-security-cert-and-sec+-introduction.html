TryHackMe — Networking

Notes

Subnetting splits a network up into smaller segments, which allows more than 254 devices on one network and creates separation between them.

ARP matches MAC addresses to IP addresses. IP addresses are not static; MAC addresses are. The protocol broadcasts a request on the local network to find which MAC owns a given IP.

DHCP assigns IP addresses automatically using a four-step process: Discover, Offer, Request, Acknowledge (DORA).

DNS translates domain names into IP addresses. Domain hierarchy goes from right to left: TLD (.com), then second-level domain (tryhackme), then subdomain (admin.tryhackme.com). Record types: A (IPv4), AAAA (IPv6), CNAME (alias), MX (mail), TXT (metadata).

The OSI model is a 7-layer framework for understanding how data moves through a network. Each layer has a specific job. Data encapsulates as it moves down the stack, picking up headers at each layer, and de-encapsulates going back up on the receiving end.

TCP is connection-based and uses a three-way handshake (SYN, SYN-ACK, ACK). It's reliable but slower. UDP has no handshake and no acknowledgement — it's faster and used for things like streaming and voice calls where some loss is acceptable.

Ports are numbered 0 to 65535. Ports below 1024 are well-known. Applications use standard ports so devices know what to expect: HTTP is 80, HTTPS is 443.

Port forwarding opens specific ports on a router to make internal services reachable from outside the network. Firewalls still control whether traffic is actually allowed through those ports.

Firewalls filter traffic based on where it's coming from, where it's going, what port it's using, and what protocol it's using.

VPNs create an encrypted tunnel between devices over the internet so they can communicate securely as if they were on the same private network.

Routers connect networks and pass data between them. Switches connect devices within a network — Layer 2 switches use MAC addresses, Layer 3 switches can also route using IP. VLANs allow devices on the same physical switch to be virtually separated into isolated networks.

TryHackMe — How the Web Works

Notes

DNS is how website addresses get translated into IP addresses. When you visit a site your computer uses DNS to find the server's IP before anything else happens.

HTTP is the protocol browsers use to communicate with web servers. HTTPS is the secure version — data is encrypted so it can't be read in transit, and you get assurance you're talking to the right server.

URLs are instructions for how to access a resource on the internet. HTTP methods define what you're doing: GET retrieves data, POST submits it, PUT updates it, DELETE removes it.

Request headers carry extra information from the client to the server: Host tells the server which site you want, User-Agent identifies your browser, Cookies carry session data. Response headers come back from the server: Set-Cookie stores data on your machine, Content-Type tells the browser what it's receiving.

Cookies are small pieces of data stored in the browser. Because HTTP is stateless, cookies let the server remember who you are between requests.

Load balancers sit in front of multiple servers and distribute incoming traffic between them. They use round-robin (taking turns) or weighted (based on current load) methods and perform health checks to stop sending traffic to servers that are down.

CDNs host static files like images, JavaScript, and CSS on servers distributed around the world and serve them from whichever node is closest to the user.

WAFs protect web servers by analysing requests for common attack patterns, checking whether traffic looks like a real browser, and rate-limiting requests from a single IP.

Web servers like Apache, Nginx, and IIS listen for connections and serve files from a root directory. One server can host multiple websites using virtual hosts, which work by reading the hostname from the HTTP request and routing it to the right site.

Static content never changes. Dynamic content is generated per request by backend code (PHP, Python, Node.js) pulling from a database. Everything visible in the browser is frontend; everything behind it is backend.

Sensitive data exposure happens when developers leave credentials or private information in the site's frontend source code. It's worth checking page source during any security assessment.

HTML injection is a client-side vulnerability where unfiltered user input gets displayed on the page. If a site doesn't sanitize what the user enters, an attacker can inject HTML and change what other users see. User input should never be trusted.

CompTIA Security+ (SY0-701) — Study Plan

The TryHackMe Pre-Security path built a working foundation in networking and web concepts that maps directly into Security+ preparation. The web module covers Domain 2 (Threats, Vulnerabilities, and Mitigations) through HTML injection, sensitive data exposure, and WAF usage.
