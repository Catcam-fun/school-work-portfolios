Lab 2: Detecting the Use of Steganography in Forensic Evidence

In this lab, I looked for and recovered hidden information from files using steganography tools. The case continued with Beverly Gates, and the task was to search her drive image for hidden communication. The challenge was that the evidence was not immediately obvious. Some of the files looked like normal images or audio files, but they contained hidden data. This made the lab different from looking through documents or emails because the important information was concealed inside other files.

In the first part of the lab, I used Paraben’s E3 to search the suspect’s drive image for steganography tools. This was done through a hash comparison with a hash database. This was important because finding steganography tools on a drive can suggest that hidden files or messages may also be present. Even if the software has been renamed, hashing can still be used to detect it. This showed me that hashing is useful for more than just verifying evidence integrity.

The next part of the lab involved using StegExpose to scan exported image files. This tool helped identify suspicious images that appeared to contain hidden data. I then used OpenPuff, S Tools, Xiao, and OpenStego to extract the hidden information from those files. The hidden data included location information, names, phone numbers, special instructions, and maps. I also found suspicious files like chicago.bmp and chicago1.bmp and estimated the amount of hidden data inside them.

One thing I found interesting was that the carrier files looked normal at first. An image or audio file may not seem suspicious, but it can still be used to hide data. This showed that investigators should not rely only on file names or appearance when reviewing evidence. It also showed how steganography can be used for hidden communication in a criminal investigation.

What I learned from this lab is that data can be hidden inside ordinary looking files, and investigators need specific tools to detect and extract it. I also learned that it is important to document both the suspicious file and the extracted contents. The file shows where the hidden data came from, while the extracted information explains why it may matter to the investigation.
