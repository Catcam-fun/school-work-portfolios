Lab 5: Conducting Forensic Investigations on Windows Systems

Click here to Download Conducting Forensic Investigations on Windows Systems (4e) - Maxwell King (1).pdf

This lab involved collecting forensic evidence from a live Windows machine and a Windows drive image. The challenge was that useful evidence in Windows can be found in many different places. Some data can be found through live system tools, some through command line utilities, and some through the registry or drive image artifacts. Instead of searching for only one file, this lab involved gathering smaller pieces of data and learning how they can support an investigation.

In the first part of the lab, I used Task Manager and Resource Monitor to collect information. Task Manager showed running processes and the file paths connected to them, while Resource Monitor showed listening ports and network connections. This was helpful because live analysis can show what is currently happening on the system. I also used fsutil to get details about the C: drive and the USN journal. The USN journal was interesting because it can provide information about file system activity, including file creation, deletion, and modification.

In the second part of the lab, I used Registry Editor to explore important locations in the Windows registry. I checked the Windows installation time, default network interface values, Winlogon keys, ShellBags, and RecentDocs. This showed that a lot of user and system activity can be tracked through the registry. RecentDocs can reveal files that were viewed, while ShellBags can reveal folders that were opened. These artifacts can help build a history of activity on the system.

In the applied section, I used Paraben’s E3 to examine a Windows hard drive image. I reviewed sorted files, suspicious image files, shortcut files, installed programs, VPN evidence, user folders, browser history, and keyword searches. Shortcut files were helpful because link files can show where a file was located on the computer and whether it may have been opened. Browser history and keyword searches were also useful because they can show user intent or suspicious activity.

What I learned most is that Windows forensics is not just about files. Windows creates many artifacts that can show what a user did, what programs were installed, what network connections existed, and what files may have been accessed. I also learned the difference between live analysis and drive image analysis. Live analysis shows current activity, while a drive image allows the investigator to review the system safely without changing the original evidence. This lab helped me understand why Windows computers are such a common source of forensic evidence.
