Lab 3: Recovering Deleted and Damaged Files

Click here to Download Recovering Deleted and Damaged Files (4e) - Maxwell King (1).pdf

The purpose of this lab was to recover deleted and damaged files from different file systems. The case involved intellectual property theft, where suspects may have removed evidence from their computers. The challenge was understanding that when a file is deleted, the data is not always completely erased. The file reference may be removed, but the data can still remain on the drive until that space is reused or overwritten.

During the first part of the lab, I used Paraben’s E3 to recover deleted files from an NTFS drive image. E3 located recovered files in the Trash folder, and I examined those files for patent and product design information. I also looked at the Windows Recycle Bin. This was helpful because the Recycle Bin can still hold files that the user intended to delete but did not fully remove. It also showed that deleted files can still contain useful metadata and original file information.

In the second part of the lab, I used Autopsy to search an Ext4 Linux drive image. Autopsy organized the deleted files in a way that made them easier to view and extract. I found another patent file on the Linux drive image, which showed that deleted evidence can be recovered from more than just Windows systems. This part helped me understand the differences between forensic tools, while also showing that the main goal is the same: find, recover, and preserve the evidence.

Later in the lab, I used PhotoRec on a Linux system to recover data through data carving. This was different because data carving relies more on file signatures instead of normal file system references. I recovered compressed files and backup files from a RAR archive. In the challenge section, I recovered files from FAT32 and APFS drive images, which showed that different file systems may require different recovery methods.

I learned that file recovery depends on how the file was deleted, what file system was used, and whether the data has been overwritten. I also realized that deleted data can be important because people may delete files specifically to hide them. This lab was a good reminder that “gone” does not always mean gone, and investigators should examine deleted data before assuming it is lost forever.
